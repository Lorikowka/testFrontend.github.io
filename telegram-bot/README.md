# 🤖 Telegram Бот — Админ-панель

Полнофункциональный Telegram-бот для управления записями психолога Екатерины Князьковой.

## Возможности

### 📅 Управление расписанием
- **Сегодня** — все записи на текущий день
- **Завтра** — записи на завтра
- **Эта неделя** — расписание с группировкой по дням
- **Все записи** — полный список будущих сеансов

### 👥 Управление записями
- ✅ Отметить сеанс как проведённый
- ❌ Отменить сеанс
- 📋 Посмотреть полные детали записи

### 📊 Статистика
- Общее количество сеансов
- Ожидаемый и фактический доход
- Записи на сегодня / завтра
- Доход за неделю

### 📤 Экспорт в календари
- **Эта неделя** (.ics файл)
- **Следующая неделя** (.ics)
- **Весь месяц** (.ics)
- **Все записи** (.ics)
- Инструкции по импорту в **Google Calendar** и **Яндекс Календарь**

### 🌅 Утренние сводки
- Автоматически в 8:00 отправляет расписание на день

### 🔔 Напоминания
- За 48 часов до сеанса
- Проверка каждый час

## Установка

### 1. Настройка окружения

```bash
# Скопируйте пример .env
copy .env.example .env

# Отредактируйте .env:
# TELEGRAM_BOT_TOKEN — токен от @BotFather
# TELEGRAM_ADMIN_ID — ваш ID от @userinfobot
# BACKEND_URL — URL backend (http://localhost:1488)
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Запуск

```bash
python bot.py
```

Или запустите `start-bot.bat`

## Как использовать ICS файлы

### Google Calendar
1. Скачайте .ics файл из бота
2. Откройте [calendar.google.com](https://calendar.google.com)
3. Настройки → Импорт и экспорт → Импортировать
4. Выберите скачанный .ics файл

### Яндекс Календарь
1. Скачайте .ics файл из бота
2. Откройте [calendar.yandex.ru](https://calendar.yandex.ru)
3. ⚙️ → Импорт календаря
4. Загрузите .ics файл

### Подписка на календарь (автообновление)
Для автообновления нужно разместить .ics файл на веб-сервере и добавить URL в календарь:
- **Google**: Настройки → Добавить календарь → По URL
- **Яндекс**: ⚙️ → Подписка на календарь → Вставить URL

## Структура базы данных

Бот работает напрямую с SQLite базой backend:
- `backend/data/payments.db` — платежи
- `backend/data/payments.db` → таблица `sessions` — записи

## Troubleshooting

### Бот не запускается
1. Проверьте токен в `.env`
2. Убедитесь, что БД существует
3. Проверьте `TELEGRAM_ADMIN_ID`

### Нет записей
- Убедитесь, что backend запущен и есть тестовые платежи в MOCK режиме
